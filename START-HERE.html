What you'll end up with

A private web app on its own link — add it to your phone's home screen and it feels like a real app. Only people who know your PIN can open it. Before you start — the honest details

⏱️ Time

About 30–45 minutes, mostly clicking through account setup.

💵 Cost

Free. Every service here has a generous free tier. No credit card needed.

🔒 Security

Protected by a shared PIN — fine for a small trusted group, but convenience-level. Don't reuse an important PIN.

🤖 Privacy

Text & photos go to Gemini to be read; on the free tier Google may use them to improve models. Photos aren't stored by the app.

1 Unzip the folder

If you're reading this file, you likely already did! You should have a folder called money-tracker containing a file named package.json and folders named app, lib, and scripts.

  • Put this folder somewhere safe, like your Desktop.
  • Don't rename or delete anything inside it — you'll upload the whole folder to GitHub in Step 6.
✅
That's the whole step. You never need to open or edit any of the code files.

2 Create your Google Sheet

This is where your expenses get stored.

  1. Go to sheets.new — this instantly makes a new blank sheet.
  2. Name it: click "Untitled spreadsheet" at the top-left and type something like Expenses.
  3. Add the header row. Click the very first cell, A1, and paste this exact line — it spreads across the first six cells:
Date    Person    Amount    Merchant    Category    Notes
💡
The gaps are Tab characters. The Copy button copies them correctly — just paste into cell A1. If everything lands in one cell instead, press Ctrl+Z and type each word into A1, B1, C1… pressing Tab between them.

Your top row should read: Date | Person | Amount | Merchant | Category | Notes

Copy your Sheet ID

Look at the web address at the top of your browser. The long jumble in the middle is your Sheet ID — select just that part (between /d/ and /edit) and copy it somewhere safe. You'll need it in Step 7.

Keep this sheet's browser tab open — you'll share it in Step 5.

3 Get a free AI key (Google Gemini)

This gives your app its brain for reading expenses.

  1. Go to aistudio.google.com and sign in with your Google account.
  2. Click "Get API key" (a key icon in the left menu).
  3. Click "Create API key", then choose "Create API key in new project".
  4. A long key appears — it may start with AIza or AQ. (both are valid). Click Copy and save it. This is your GEMINI_API_KEY.
🔑
Treat this key like a password — anyone with it can use your free AI quota.

4 Create a "robot" account for the sheet

Your app can't log into Google as you. Instead it uses a service account — a robot identity allowed to write to just your one sheet. This looks technical but it's only clicking. Follow it exactly.

  1. Go to console.cloud.google.com and sign in. If asked to accept terms, do so — you will not be charged and don't need any free trial.
  2. Pick the right project. At the very top, next to "Google Cloud", click the project dropdown and choose the project made in Step 3 (often named "Gemini API" or "My First Project"). Any project works.
  3. Turn on the Sheets API: in the top search bar type Google Sheets API, click the result, then click the blue "Enable" button.
  4. Create the service account: search Service Accounts (under "IAM & Admin") → "+ Create service account" → name it expense-bot → "Create and continue" → skip the optional access steps → "Continue" → "Done".
  5. Create its key file: click your new expense-bot account → the "Keys" tab → "Add key" → "Create new key" → choose JSON → "Create". A .json file downloads.
  6. Open that downloaded file in any text editor (Notepad, TextEdit, or your browser). Find and copy these two values:
🔒
That JSON file is like a password. Keep it private, and delete it once you've finished Step 7.
  • "client_email" — ends in …iam.gserviceaccount.com → your GOOGLE_SERVICE_ACCOUNT_EMAIL.
  • "private_key" — a long block starting -----BEGIN PRIVATE KEY-----. Copy everything between the quotation marks, including the BEGIN/END lines → your GOOGLE_PRIVATE_KEY.

5 Share your sheet with the robot

The robot exists but can't see your sheet yet. Let's fix that.

⚠️
This is the single most-forgotten step. Skip it and the app can't save or read anything (you'll get a "permission denied" error).
  1. Go back to your Google Sheet from Step 2.
  2. Click the green "Share" button (top-right).
  3. Paste the robot's email — your GOOGLE_SERVICE_ACCOUNT_EMAIL ending in …iam.gserviceaccount.com.
  4. Set its role to "Editor" (not Viewer).
  5. Uncheck "Notify people" — the robot has no inbox.
  6. Click "Share" / "Send".

6 Put the code on GitHub

Upload this folder to GitHub so Vercel can find it — all in your browser.

  1. Go to github.com/new (sign in if needed).
  2. Repository name: money-tracker.
  3. Set it to Private.
  4. Leave every other box unchecked (no README, no .gitignore, no license) — the upload needs an empty repository.
  5. Click "Create repository".
  6. On the next page, click the small link "uploading an existing file".
  7. Open your money-tracker folder, select everything inside it (Ctrl/Cmd+A) — the app, lib, scripts folders and the loose files — and drag them all onto the upload area.
  8. Scroll down and click the green "Commit changes".
💡
Give the upload a moment — it keeps the folder structure. You should see app/, lib/, scripts/, and package.json in the list. The tutorial files ride along harmlessly.

7 Put it online with Vercel

Now it becomes a real, live website.

  1. Go to vercel.com/new. Sign in with "Continue with GitHub", same account as Step 6.
  2. Find money-tracker in your repo list and click "Import". (Not listed? Click "Adjust GitHub App Permissions" and grant access.)
  3. Before clicking Deploy, expand "Environment Variables" and add each row below — type the Name, paste the Value, click Add, repeat.
Name (click to copy)Value — where it comes from
APP_PINA number you pick, e.g. 2468. The PIN you'll both type to open the app.
GEMINI_API_KEYThe key from Step 3 (it may start with AIza or AQ.).
GOOGLE_SERVICE_ACCOUNT_EMAILThe …iam.gserviceaccount.com email from Step 4.
GOOGLE_PRIVATE_KEYThe long -----BEGIN PRIVATE KEY-----… block from Step 4. Paste exactly as-is (multiple lines are fine).
GOOGLE_SHEET_IDThe long ID from your sheet's URL, Step 2.
SHEET_NAMESheet1 — the tab name at the bottom-left of your sheet. Leave as Sheet1 unless you renamed the tab.
PEOPLEEveryone's names, separated by commas — e.g. Alex,Sam or Alex,Sam,Jordan. Add as many as you like; the first name is the default.
💡
You can skip GEMINI_MODEL — the app defaults to gemini-2.5-flash, which is free and fast.
  1. Once they're all added, click the big "Deploy" button.
  2. Wait 1–2 minutes. When it's done you'll see confetti 🎉 and a link like money-tracker-xxxx.vercel.app. Click "Continue to Dashboard" — your link is shown under "Domains".

8 Open it and share it

  1. Open your …vercel.app link — on your computer or straight on your phone.
  2. Type the PIN you chose (APP_PIN) and tap Unlock.
  3. Tap your name at the top so entries are attributed to you (remembered on your device).
  4. Try it: type 12 coffee and tap Log expense. Check that a row appears in your Google Sheet and the "This Month" total updates.
  5. Add it to your home screen: iPhone (Safari) → Share icon → "Add to Home Screen". Android (Chrome) → ⋮ menu → "Add to Home screen".
  6. Send the link and PIN to the other person so they can do the same — they just tap their name once.
🎉
You're done — you built and shipped a real web app.

Troubleshooting

The app usually tells you what's wrong in a red box. Match it here:

See the symptom → fix table
What you seeFix
"permission denied" / 403The sheet isn't shared with the robot. Redo Step 5 (share as Editor). Also confirm the Sheets API is enabled (Step 4.3).
"private key" / "not a valid private key"The GOOGLE_PRIVATE_KEY got mangled. In Vercel → Settings → Environment Variables, re-paste the whole block including the BEGIN/END lines, then redeploy.
"Spreadsheet not found" / 404Wrong GOOGLE_SHEET_ID. Recopy just the middle of the sheet URL (Step 2), update Vercel, redeploy.
"rate limit" / 429Hit Gemini's free per-minute limit. Wait a minute and try again.
Totals show $0 / emptyMake sure row 1 is the headers (Step 2) and expenses are below. The summary only counts the current month (Pacific time).
Shows "Alex / Sam"You didn't set PEOPLE in Vercel, or misspelled it. Fix (comma-separated names) and redeploy.
"Wrong PIN" but it's rightCheck APP_PIN in Vercel for stray spaces. Env changes need a redeploy.

How to redeploy: Vercel → your project → Deployments tab → ⋯ on the most recent → Redeploy. Environment-variable changes only apply to new deployments.

Frequently asked

Is it really free?

Yes. Vercel's Hobby plan, Google Sheets, and Gemini's free tier all cost nothing at this scale, and none of the steps ask for a credit card.

How many people can use it?

As many as you list in the PEOPLE variable — two, three, or more. Everyone shares one link and PIN, and each person taps their own name. To add someone later, edit PEOPLE in Vercel and redeploy.

Can I edit the sheet by hand?

Absolutely — add, fix, or delete rows directly and the totals follow. Keep dates as YYYY-MM-DD (or real date cells).

Where are my receipt photos stored?

Nowhere. They're sent to the AI to be read, then discarded. Only the resulting text is saved.

Advanced (optional) — run it on your own computer

You don't need this for the steps above. If you're comfortable with a terminal:

  1. Install Node.js (version 20+).
  2. Copy .env.example to a new file named .env.local and fill in the same values from Step 7.
  3. Run npm install, then npm run setup:sheet (writes the header row for you), then npm run dev, and open http://localhost:3000.
